أنت Senior Expo/React-Native Engineer داخل نفس المشروع. 
ممنوع تغيير أي ميزة قديمة أو كسر شاشات موجودة. فقط أضف شاشات/تعديلات مطلوبة.
هدفنا إضافة: PharmacyPickerScreen + PharmacyRouteScreen (خريطة + مسار) مع مميزات كثيرة وواجهة فخمة.
المشروع RTL عربي/إنجليزي، ولازم يدعم وضع التطبيق (Light/Dark) وليس فقط وضع الجهاز.

=====================================================
(1) تدفق العمل (Flow)
=====================================================
- بعد ما يطلع نتائج الصيدليات لعلاج معين (من AI search أو بحث يدوي):
  افتح PharmacyPickerScreen (اختيار الصيدلية).
- من تختار صيدلية:
  افتح PharmacyRouteScreen (خريطة + مسار + مميزات).

=====================================================
(2) الخريطة: OpenStreetMap + وضع نهاري/ليلي حسب التطبيق
=====================================================
- لا تستخدم Google Maps API.
- استخدم react-native-maps + UrlTile.
- لازم تدعم 2 ستايلات tiles حسب ثيم التطبيق:
  Light tiles:
    https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png
  Dark tiles (بدون مفتاح) استخدم واحد من التالي (اختيار واحد شغال):
    https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png
  (إذا هذا ما اشتغل لأي سبب، fallback لخيار ثاني dark tile مجاني)
- استخرج حالة الثيم من نظام التطبيق الموجود عندي (useTheme أو AppContext أو أي ثيم عندي)
  وبدّل UrlTile تلقائياً عند تغيير الثيم داخل التطبيق (بدون إعادة تشغيل).

=====================================================
(3) مميزات ضخمة داخل PharmacyRouteScreen
=====================================================
A) Route + معلومات
- جلب موقعي عبر expo-location (مع handling للأذونات + رسالة واضحة إذا رفض).
- جلب المسار عبر OSRM endpoint (بدون مفتاح):
  https://router.project-osrm.org/route/v1/driving/{lon1},{lat1};{lon2},{lat2}?overview=full&geometries=polyline6
- فك polyline6 وارسم Polyline.
- اعرض داخل Bottom Sheet:
  اسم الصيدلية + العنوان + رقم الهاتف (إن وجد)
  المسافة (كم) + الزمن (دقيقة) + سرعة تقريبية (اختياري)
- اعرض شارة “GPS ضعيف” إذا accuracy سيئة (إذا توفرت).

B) عناصر خريطة “حلوة جداً”
- Marker موقعي: استخدم Custom Marker (View دائري) بلون حسب الثيم + Glow خفيف (Shadow).
- Marker الصيدلية: Custom Marker بأيقونة صيدلية/موقع (Feather/MaterialIcons) حسب المكتبات الموجودة.
- Polyline للمسار: لون واضح حسب الثيم + StrokeWidth مناسب + خط outline خفيف إن أمكن.
- أضف “Compass” بسيط: زر يرجع اتجاه الخريطة للشمال (reset camera bearing).
- أضف زر Zoom + و Zoom - (animated camera).
- أضف زر “تتبعني” On/Off:
  عند On: يحدث موقعي كل 3-5 ثواني ويحرّك الكاميرا عليّ.
  وعند Off: يوقف التتبع.
- أضف “إعادة حساب المسار” عند تغيّر موقعي (إذا التتبع On) لكن مع rate limit:
  لا تعيد fetch أكثر من مرة كل 15 ثانية.

C) Bottom Sheet فاخر
- Bottom Sheet فوق الخريطة بتصميم Luxury:
  rounded كبير + blur/opacity حسب الثيم + padding ممتاز.
- قابل للسحب (إن كان عندي مكتبة bottom sheet جاهزة)؛ إذا لا:
  استخدم Card ثابت لكن منظم.
- داخل الـ sheet:
  (1) عنوان الصيدلية + نجمة (favorite placeholder)
  (2) مسافة/وقت على شكل Pills
  (3) أزرار كبيرة:
      - “فتح بويز”
      - “خرائط”
      - “اتصال”
      - “واتساب”
      - “معلومات الصيدلية” (ينقل للواجهة القديمة)
- زر “مشاركة الموقع” (Share) يشارك رابط Google Maps web مع الإحداثيات.

D) حالات خاصة وموثوقية
- Loading state: "دا أحسب المسار..." + ActivityIndicator.
- Error state: إذا فشل OSRM أو ماكو route:
  - اعرض رسالة
  - اعرض زر Retry
  - وخلي الخريطة تبقى تظهر Markers بدون Route
- إذا المستخدم ما مفعل GPS:
  - اعرض تنبيه
  - زر يفتح Settings (إن ممكن على المنصة).
- إذا الانترنت ضعيف:
  - timeout للـ fetch (مثلاً 10 ثواني) + retry واحد.

E) أداء (Performance)
- Cache route داخل الذاكرة (Map key = from/to rounded 5 decimals).
- استخدم useMemo/useCallback لمنع re-renders.
- لا تحمل icons/صور كبيرة.

=====================================================
(4) PharmacyPickerScreen مميزات حلوة
=====================================================
- Search input يفلتر الصيدليات بالاسم.
- Sort options:
  1) الأقرب (requires compute distance بين موقعي والصيدليات، لو الاذن موجود)
  2) أبجدي
- Card لكل صيدلية:
  الاسم + (المسافة إذا متاحة) + زر “مسار” وزر “معلومات”.
- UI يدعم RTL وLight/Dark.

=====================================================
(5) Navigation + الدمج بالشاشة الحالية
=====================================================
- أضف الشاشتين للـ Stack Navigator.
- عدّل شاشة نتائج البحث الحالية:
  onPress الافتراضي يفتح PharmacyPickerScreen بدل تفاصيل الصيدلية مباشرة.
  وداخل PharmacyPickerScreen يوجد زر “معلومات” يرجع للتفاصيل القديمة.

=====================================================
(6) التوافق مع المشروع
=====================================================

- استخرج isDark من ثيم التطبيق الحقيقي (مو ثيم الجهاز فقط).
- لا تضف مكتبات جديدة ثقيلة. فقط استخدم الموجود (react-native-maps, expo-location, icons).
- TypeScript مضبوط، بدون any إلا للضرورة.
