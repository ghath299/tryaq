====================================
اسم المشروع
====================================
منصة صحية ذكية

====================================
فكرة المشروع العامة
====================================
منصة صحية تربط بين:
- المستخدم (المريض)
- الطبيب
- الصيدلاني
- المدير (Admin)

توفر:
- البحث عن الأطباء
- طلب حجز طبيب
- البحث عن الأدوية
- البحث عن الصيدليات
- حجز أو توصيل الأدوية
- إدارة طابور المرضى يدويًا للأطباء
- إدارة الأدوية للصيدليات
- ذكاء اصطناعي للتعرف على الأدوية
- نظام تحقق صارم للأطباء والصيادلة
- واجهات تعتمد على الأنيميشن بشكل أساسي

====================================
الأدوار (Roles)
====================================
1. مستخدم (مريض)
2. طبيب
3. صيدلاني
4. مدير النظام

====================================
التسجيل وتسجيل الدخول
====================================
- إدخال الاسم الثلاثي (حروف عربية فقط، بدون رموز أو كلمات غير لائقة)
- إدخال رقم الهاتف
- تفعيل الموقع إجباري ولا يمكن التخطي
- إرسال كود تحقق (SMS أو WhatsApp)
- بعد التحقق يختار المستخدم دوره:
  - مستخدم
  - طبيب
  - صيدلاني

====================================
قواعد الواجهة العامة
====================================
- زر القائمة الجانبية ☰ موجود في جميع الشاشات بدون استثناء
- محتوى القائمة الجانبية يتغير حسب نوع الحساب
- دعم الوضع الليلي والنهاري
- دعم اللغة العربية والإنكليزية
- التصميم يعتمد على الأنيميشن
- لا توجد أزرار تقليدية مربعة
- انتقالات ناعمة وتفاعلات صغيرة (Micro-interactions)

====================================
الشريط السفلي (للمستخدم فقط)
====================================
4 أزرار فقط:
1. الرئيسية
2. الأطباء
3. العلاجات
4. الصيدليات

(لا يوجد زر حساب)

====================================
القائمة الجانبية (حسب الدور)
====================================

عناصر ثابتة لجميع الأدوار:
- تغيير اللغة
- وضع النهار / الليل
- حول التطبيق
- سياسة الخصوصية
- تسجيل الخروج

------------------------------------
قائمة المستخدم
------------------------------------
رأس القائمة:
- صورة حساب
- الاسم الثلاثي
- رقم الهاتف
- المحافظة فقط

العناصر:
- حجوزاتي
- طلباتي

------------------------------------
قائمة الطبيب
------------------------------------
رأس القائمة:
- اسم الطبيب
- الاختصاص
- حالة التوثيق

العناصر:
- مرضى اليوم
- الدوام والإجازات
- سجل العمل (آخر 15 يوم فقط)
- ملف الطبيب

------------------------------------
قائمة الصيدلاني
------------------------------------
رأس القائمة:
- اسم الصيدلية
- حالة التوثيق

العناصر:
- إدارة العلاجات
- الطلبات الواردة
- سجل الصيدلية (آخر 15 يوم فقط)
- ملف الصيدلية

====================================
واجهة المستخدم (المريض)
====================================

الرئيسية:
- سلايدر إعلانات متحرك
- ترويج أطباء
- ترويج صيدليات
- ترويج أدوية
- أحداث صحية

------------------------------------
قسم الأطباء:
التسلسل الإجباري:
1. بحث عام باسم الطبيب (من أول حرف، كل العراق)
2. اختيار التخصص (إجباري)
3. اختيار المحافظة (إجباري)
4. بحث مخصص داخل التخصص + المحافظة
5. قائمة الأطباء:
   - الاسم
   - الاختصاص
   - القضاء / الناحية
   - المسافة
6. صفحة الطبيب:
   - خريطة حية
   - معلومات العيادة
   - أيام وأوقات الدوام
   - صور العيادة
7. طلب الحجز:
   - الاسم الثلاثي (إجباري)
   - العمر (اختياري)
   - ملاحظات (اختياري)

------------------------------------
قسم العلاجات:
يوجد خياران فقط:

1. البحث النصي:
- كتابة اسم العلاج
- تظهر الشركات
- اختيار شركة
- اختيار محافظة
- تظهر الصيدليات المتوفرة مع:
  - المنطقة
  - المسافة
  - التوصيل
- صفحة الصيدلية + خريطة
- حجز أو توصيل

2. الذكاء الاصطناعي:
- رفع صورة العلاج
- الذكاء الاصطناعي يتعرف على:
  - اسم العلاج
  - الشركة
  - الاستخدام
- الاسم يكون:
  - قابل للنسخ
  - أو بالضغط عليه يُكتب تلقائيًا في البحث

------------------------------------
قسم الصيدليات (للمستخدم):
- البحث عن صيدلية
- فلترة حسب المحافظة
- صفحة الصيدلية:
  - خريطة
  - معلومات
  - قائمة الأدوية
- حجز أو توصيل

====================================
تسجيل الطبيب
====================================
- الاسم الكامل
- الاختصاص
- معلومات العيادة
- رفع مستند إلزامي يثبت أنه طبيب
- تثبيت موقع العيادة يدويًا على الخريطة
- الحالة: قيد المراجعة

====================================
واجهة الطبيب بعد الموافقة
====================================
- لوحة تحكم (ملخص اليوم)
- طابور المرضى يدوي (بدون نظام وقت)
- البحث باسم المريض الأول (حجوزات اليوم فقط)
- أزرار:
  - حضر
  - تم
- الدوام والإجازات
- ملف الطبيب
- سجل العمل:
  - آخر 15 يوم فقط
  - تفاصيل دقيقة بالساعة والدقيقة
- حذف تلقائي لأي سجل أقدم من 15 يوم
- السجل يظهر للطبيب وللمدير

====================================
تسجيل الصيدلاني
====================================
- الاسم الكامل
- معلومات الصيدلية
- رفع مستند إلزامي يثبت أنه صيدلاني
- تثبيت موقع الصيدلية يدويًا
- الحالة: قيد المراجعة

====================================
واجهة الصيدلاني بعد الموافقة
====================================
- إدارة العلاجات (من مكتبة مركزية فقط)
- تفعيل / تعطيل الأدوية
- الطلبات الواردة
- سجل الصيدلية (آخر 15 يوم فقط)
- ملف الصيدلية

====================================
الذكاء الاصطناعي للصيدليات
====================================
- التعرف على الدواء من الصورة
- اقتراح أدوية شائعة
- التنبؤ بنفاد الدواء
- التنبؤ بعودة الدواء
- الذكاء لا يغيّر الحالة تلقائيًا
- الصيدلاني يقرر دائمًا

====================================
واجهة المدير
====================================
- مراجعة الأطباء
- مراجعة الصيادلة
- مشاهدة المستندات
- مشاهدة المواقع على الخريطة
- الموافقة / الرفض / التعليق
- إدارة مكتبة الأدوية
- عرض السجلات (آخر 15 يوم فقط)
- مراقبة اقتراحات الذكاء الاصطناعي

====================================
قواعد البيانات
====================================
- حذف تلقائي لأي سجل أقدم من 15 يوم
- لا يوجد أرشيف قديم
- كل العمليات الحساسة مسجلة
- صلاحيات صارمة حسب الدور

====================================
المتطلبات التقنية
====================================
- تطبيق موبايل متكامل
- Backend + Database
- نظام أدوار
- خرائط
- ذكاء اصطناعي للصور والتنبؤ
- أنيميشن عالي الأداء
