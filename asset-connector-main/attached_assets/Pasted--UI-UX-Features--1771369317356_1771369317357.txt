أريدك تصلّح مشاكل UI/UX الحالية في تطبيق المريض فقط بدون ما تغيّر Features أو تضيف شغل جديد.
ركز فقط على ترتيب العرض + النصوص + RTL + أنميشن بسيط واحترافي.

1) مشكلة رمز التحقق OTP (RTL)

داخل شاشة التحقق من الرقم:

حالياً مربعات إدخال الكود (OTP) تظهر من اليمين لليسار.

المطلوب: تظهر من اليسار لليمين (LTR) مثل الطبيعي.

مع بقاء باقي الشاشة عربي (RTL).

تأكد أن التنقل بين الخانات Auto-focus صحيح.

2) شاشة الذكاء الاصطناعي (AI Results) مخربطة

داخل شاشة Medicines / AI Scan:

مشاكل:

مكتوب "نتائج التحاليل" وهذا غلط (مو تحليل طبي).

التسميات غير واضحة:

"اسم الدواء"

"دقة التحليل"

"النص المستخرج"

"التصنيف"

"الإرشادات"

ترتيب النتائج غير احترافي.

زر "مسح النتيجة" غير واضح ومكانه غلط.

تصميم البطاقة (Card) ضعيف.

المطلوب:

تغيير العناوين والتسميات إلى صياغة صحيحة للمريض:

"نتائج التعرف على الدواء"

"اسم الدواء"

"نسبة الثقة"

"النص المستخرج من الصورة"

"الفئة"

"ملاحظات/إرشادات"

إعادة تصميم العرض:

Card مرتبة

spacing مضبوط

نفس الثيم الداكن

أنميشن بسيط عند ظهور النتيجة (Fade + Slide)

زر "مسح النتيجة" يتحول إلى:

"مسح النتيجة"

يكون صغير ومرتب داخل الكارد (يمين أو أسفل)

الأهم: العرض يكون واضح ومفهوم للمستخدم.

3) شاشة تفاصيل الصيدلية + الشات

داخل:

client/screens/PharmacyDetailScreen.tsx

مشاكل:

ترتيب الاسم + المحافظة + الرقم + الأزرار غير مرتب.

الشات شكله فوضوي.

لازم يكون composer واضح.

لازم يكون شكل الرسائل مرتب (bubble + alignment + time).

عند فتح شاشة الصيدلية لازم يظهر transition احترافي (fade/slide).

المطلوب:

إعادة ترتيب Header:

اسم الصيدلية واضح

المحافظة تحت الاسم

رقم الهاتف واضح

ترتيب أزرار:

اتصال

واتساب

نسخ الرقم
بشكل Grid أو row مرتب.

الشات:

FlatList فقط (بدون ScrollView)

رسائل المريض يمين، الصيدلية يسار

bubble styling بسيط وراقي

auto scroll يعمل

optimistic UI يبقى

4) مشكلة القائمة الجانبية Drawer

داخل Expo Go:

عند الضغط على الثلاث خطوط، القائمة الجانبية تظهر مقصوصة (نصها مو كامل).

المطلوب: إصلاح عرض الـ Drawer ليظهر كامل بالعرض.

تأكد من:

drawerStyle width مناسب

SafeAreaView

RTL صحيح

لا يوجد overflow hidden يسبب قص.

قواعد صارمة:

لا تغيّر Features.

لا تغيّر API.

لا تكسر المشروع.

اشتغل على مراحل + commit واضح.

لا تلمس واجهات الصيدلاني/الطبيب.

Testing:

npm run check:types

npm run expo:web

تأكد يعمل على Expo Go Android
