أنت تعمل داخل مشروع Expo (React Native) مفتوح عبر Expo Go. عندي مشاكل UI/UX لازم تنحل بالكامل وبشكل نظيف واحترافي. أريدك تفحص الكود وتعدل الملفات اللازمة وتكتبلي التغييرات بوضوح.

المشاكل المطلوبة:

1) Drawer/الشاشة الجانبية (Menu) لما أحوّل اللغة للعربي تصير “مقسومة/منزاحة” مثل الصورة، بينما بالإنكليزي تمام.
- المطلوب: دعم RTL بشكل صحيح:
  - تفعيل RTL عند العربية باستخدام I18nManager (forceRTL / allowRTL) بطريقة آمنة (بدون كراش) + إعادة تحميل عند الحاجة.
  - ضبط React Navigation Drawer: drawerPosition = "right" عند RTL، و drawerStyle width ثابت/نسبي صحيح.
  - منع أي transform / absolute positioning يسبب انقسام أو قص.
  - تأكد من safe areas و status bar.
- معيار القبول: بالـ Arabic drawer يطلع كامل، غير مقسوم، ومحاذاة العناصر صحيحة مثل الإنكليزي.

2) كارت “نتائج تحليل الدواء” الأسود (نتائج الذكاء الاصطناعي) يطلع مقصوص من الأسفل ومتعارض مع الشريط السفلي (tab bar / navigation).
- المطلوب:
  - خلي الكارت يدعم scroll داخلي أو يكون داخل ScrollView/BottomSheet بحيث أقدر أرفعه لفوك وأنزله.
  - عالج مشكلة التداخل مع الشريط السفلي باستخدام safe-area insets (paddingBottom).
  - إذا الكارت overlay: استخدم Modal/BottomSheet أو Animated View مع maxHeight محسوب من الشاشة.
- معيار القبول: أگدر أشوف كل المحتوى للنهاية بدون ما ينقص، وماكو تداخل ويا الشريط السفلي.

3) صفحة الصيدلية: الخريطة صغيرة جدًا + أريد زر يبدّل: "إظهار الخريطة بالأعلى" أو "تكبير/تصغير" بدل ما تكون دائمًا ظاهرة بحجم صغير.
- المطلوب:
  - خلّي map height responsive (مثلاً 220-280) + زر Expand/Collapse:
    - collapsed: خريطة صغيرة
    - expanded: خريطة كبيرة أعلى الصفحة (مثلاً 45-55% من ارتفاع الشاشة)
  - حافظ على الأداء.
- معيار القبول: زر واضح يوسّع/يقلّص الخريطة، والتصميم مرتب.

4) أزرار نسخ الرقم/واتساب/اتصال وغيرها طالعة خارج مسارها (Layout broken).
- المطلوب:
  - إعادة تنسيق صف الأزرار باستخدام flex صحيح:
    - flexDirection row
    - flexWrap عند الحاجة
    - alignItems/justifyContent مضبوط
    - استخدام gap/padding/margins
  - دعم RTL: ترتيب الأزرار ومحاذاتها بالعربي مضبوط.
- معيار القبول: الأزرار داخل الكونتينر، ما تطلع برّه، وتكون متناسقة على الشاشات المختلفة.

5) Dark/Light Mode:
- حاليًا يعتمد على وضع الجهاز فقط. أريد سويتش داخل التطبيق يبدّل (ليل/نهار) حتى لو الجهاز ثابت على وضع معيّن.
- المطلوب:
  - ThemeContext + تخزين الاختيار في AsyncStorage
  - خيار "System / Light / Dark" (يفضل) أو على الأقل Toggle Light/Dark
  - تطبيق الألوان على كل الشاشات (خصوصًا drawer, tabs, cards).
- معيار القبول: أبدّل من داخل التطبيق فورًا ويثبت الاختيار بعد الإغلاق.

6) تعارض مع أزرار التنقل في الجهاز (Android navigation bar) بسبب full screen:
- المطلوب:
  - استخدام react-native-safe-area-context
  - تطبيق paddingBottom من insets على الصفحات الحساسة + tabBarStyle
  - إذا أزرار النظام ظاهرة: خلي حد/مسافة أسفل
  - إذا اختفت: يستغل كامل الشاشة.
- معيار القبول: ماكو أي عنصر ينقص تحت أو ينضغط بسبب أزرار النظام على أي جهاز.

7) إضافات أنميشن/حركة (مهم جدًا):
- أريد تحسينات حركة احترافية بدون ثقل:
  - انتقالات بين الصفحات (stack transitions)
  - micro-interactions للأزرار (Pressable scale/fade)
  - ظهور drawer بسلاسة
  - ظهور كارت نتائج الذكاء الاصطناعي ب animation (slide up + fade)
- إذا متاح استخدم react-native-reanimated + moti أو Animated API.
- معيار القبول: الأنميشن ناعم، سريع، وما يسبب تهنيگ على Expo Go.

مطلوب منك تنفيذ الحل:
A) ابحث عن ملفات: navigation (Drawer/Stack/Tabs), screens للصيدلية، شاشة العلاجات/AI Results.
B) نفّذ تعديلات RTL + safe area + layout + theme.
C) اكتبلي تقرير مختصر: شنو الملفات اللي تغيرت + ليش + أهم الأسطر.
D) إذا احتجت مكتبات إضافية، استخدم المكتبات المناسبة لـ Expo (بدون native linking المعقّد).

ملاحظة: لا تكتفي بنصائح—نفّذ التعديل بالكود داخل المشروع.
